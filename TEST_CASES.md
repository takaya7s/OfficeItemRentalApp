# テストケースと実施手順

## 対象

- アプリ: `OfficeItemRentalApp`
- 実行環境: C# 7.3 / .NET Framework 4.8
- 保存方式: `System.Text.Json` による JSON ファイル保存
- 保存先: 実行時のカレントディレクトリ配下 `data/item-rentals.json`

## 事前準備

### 1. ビルド

```powershell
cd E:\dev\Codex\work\OfficeItemRentalApp
dotnet restore
dotnet build
```

期待結果:

- `bin\Debug\net48\OfficeItemRentalApp.exe` が生成される
- ビルドエラーが発生しない

### 2. テスト用フォルダから起動する

アプリはカレントディレクトリ配下にデータを保存するため、テスト時は専用フォルダから起動すると既存データに影響しません。

```powershell
$App = "E:\dev\Codex\work\OfficeItemRentalApp\bin\Debug\net48\OfficeItemRentalApp.exe"
$Work = Join-Path $env:TEMP "OfficeItemRentalAppManualTest"
New-Item -ItemType Directory -Force $Work | Out-Null
Push-Location $Work
& $App
```

テスト終了後:

```powershell
Pop-Location
```

テストデータを初期化したい場合は、テスト用フォルダ内の `data\item-rentals.json` を退避または削除してから再起動します。

## 共通テストデータ

| 備品ID | 備品名 | 区分 |
| --- | --- | --- |
| P001 | ノートPC-A | PC |
| PJ001 | プロジェクター1 | プロジェクター |
| O001 | HDMIケーブル | その他 |

社員名の例:

- 田中(営業部)
- 鈴木(開発部)

日付の例:

- 貸出日: `2026/07/01`
- 返却予定日: `2026/07/10`
- 期限超過確認用の返却予定日: 実施日より前の日付

## テストケース

| No | 観点 | 手順 | 期待結果 |
| --- | --- | --- | --- |
| TC-01 | 起動 | exe を起動する | メニューが表示される |
| TC-02 | 終了 | メニューで `7` を入力する | `終了します。` と表示され、アプリが終了する |
| TC-03 | 備品未登録時の一覧 | 初期状態で `4. 備品一覧表示` を選択する | `備品が登録されていません。` と表示される |
| TC-04 | 備品登録 | `1` を選択し、`P001`、`ノートPC-A`、区分 `1` を入力する | `備品を登録しました。` と表示される |
| TC-05 | 備品ID重複 | TC-04 実施後、備品登録で同じ `P001` を入力する | 備品ID入力直後に `同じ備品IDがすでに登録されています。` と表示され、備品名と区分は入力不要になる |
| TC-06 | 備品ID重複の大文字小文字 | `P001` 登録後、備品登録で `p001` を入力する | 備品ID入力直後に大文字小文字を区別せず重複扱いになる |
| TC-07 | 区分入力エラー | 備品登録時、区分に `9` を入力後、`1` を入力する | `1～3の番号を入力してください。` と表示後、再入力できる |
| TC-08 | 貸出正常 | `2` を選択し、`P001`、`田中(営業部)`、貸出日、返却予定日を入力する | `貸出を登録しました。` と表示される |
| TC-09 | 未登録備品の貸出 | `2` を選択し、存在しない `XXX` を入力する | `指定された備品IDは登録されていません。` と表示される |
| TC-10 | 二重貸出防止 | TC-08 実施後、再度 `P001` を貸し出そうとする | 貸出不可となり、現在の借主と返却予定日が表示される |
| TC-11 | 返却予定日の前後チェック | 貸出日より前の返却予定日を入力する | `返却予定日は貸出日以降の日付を入力してください。` と表示され、再入力できる |
| TC-12 | 備品一覧の貸出中表示 | 貸出中の備品がある状態で `4` を選択する | 状態が `貸出中`、借主、貸出日、返却予定日が表示される |
| TC-13 | 督促対象表示 | 返却予定日を実施日より前にした貸出を登録し、`4` を選択する | 督促列に `督促対象` と表示される |
| TC-14 | 社員別貸出状況 | `5` を選択し、`田中` を入力する | 田中さんが現在借りている備品のみ表示される |
| TC-15 | 社員別部分一致 | `5` を選択し、`営業` など名前の一部を入力する | 借主文字列に一致する貸出中備品が表示される |
| TC-16 | 社員別該当なし | `5` を選択し、存在しない社員名を入力する | `該当する貸出中の備品はありません。` と表示される |
| TC-17 | 備品別履歴表示 | `6` を選択し、`P001` を入力する | 過去と現在の貸出履歴が表示される |
| TC-18 | 未登録備品の履歴表示 | `6` を選択し、存在しない `XXX` を入力する | `指定された備品IDは登録されていません。` と表示される |
| TC-19 | 返却正常 | `3` を選択し、貸出中の `P001` と返却日を入力する | `返却を登録しました。` と表示される |
| TC-20 | 未登録備品の返却 | `3` を選択し、存在しない `XXX` を入力する | `指定された備品IDは登録されていません。` と表示される |
| TC-21 | 貸出中でない備品の返却 | 返却済み、または未貸出の備品IDで返却する | `この備品は現在貸出中ではありません。` と表示される |
| TC-22 | 返却日の前後チェック | 貸出日より前の返却日を入力する | `返却日は貸出日以降の日付を入力してください。` と表示され、再入力できる |
| TC-23 | 返却後の一覧 | TC-19 後に `4` を選択する | 対象備品の状態が `利用可` と表示される |
| TC-24 | 返却後の履歴 | TC-19 後に `6` で対象備品を確認する | 返却日が表示され、状態が `返却済み` になる |
| TC-25 | 日付形式 | 日付欄に `2026/07/10`、`2026-07-10`、`20260710` を入力する | いずれの形式も有効な日付として登録できる |
| TC-26 | 日付入力エラー | 日付欄に `abc` や `2026/99/99` を入力する | 日付形式エラーが表示され、再入力できる |
| TC-27 | データ永続化 | 備品登録と貸出後にアプリを終了し、同じフォルダから再起動する | 登録済み備品と貸出状態が保持されている |
| TC-28 | データファイル破損 | `data\item-rentals.json` に不正な JSON を保存して起動する | 読み込み失敗メッセージが表示され、異常データをそのまま処理しない |

## 代表的な実施シナリオ

### シナリオ A: 登録から返却まで

1. `1. 備品登録` で `P001 / ノートPC-A / PC` を登録する
2. `2. 貸出処理` で `P001` を `田中(営業部)` に貸し出す
3. `4. 備品一覧表示` で `貸出中` と借主が表示されることを確認する
4. `5. 社員別貸出状況の確認` で `田中` を検索する
5. `6. 備品別の貸出履歴表示` で `P001` の履歴を確認する
6. `3. 返却処理` で `P001` を返却する
7. `4. 備品一覧表示` で `利用可` になっていることを確認する
8. `6. 備品別の貸出履歴表示` で返却日が残っていることを確認する

### シナリオ B: 督促対象の確認

1. `PJ001 / プロジェクター1 / プロジェクター` を登録する
2. 返却予定日を実施日より前の日付にして貸し出す
3. `4. 備品一覧表示` を選択する
4. 督促列に `督促対象` と表示されることを確認する

### シナリオ C: 二重貸出防止

1. `O001 / HDMIケーブル / その他` を登録する
2. `O001` を `鈴木(開発部)` に貸し出す
3. 返却前に再度 `O001` を貸し出そうとする
4. 貸出不可となり、現在の借主と返却予定日が表示されることを確認する

## 判定基準

- 全テストケースの期待結果どおりに表示、登録、保存されること
- アプリが予期せず終了しないこと
- 入力エラー時に再入力できること
- 貸出中の備品が二重に貸し出されないこと
- 返却後も履歴が消えないこと
