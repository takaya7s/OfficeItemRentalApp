# テストケースと実施手順

## 対象

- GUIアプリ: `OfficeItemRentalApp`
- 既存CLI: `OfficeItemRentalApp.CLI`
- 共通ロジック: `OfficeItemRentalApp.Core`
- 実行環境: C# 7.3 / .NET Framework 4.8
- 保存方式: `System.Text.Json` による JSON ファイル保存
- GUI版保存先: GUIアプリ実行フォルダ配下 `data/item-rentals.json`

## 事前準備

```powershell
dotnet restore .\OfficeItemRentalApp.sln --configfile NuGet.config
dotnet build .\OfficeItemRentalApp.sln --no-restore -m:1 -p:UseSharedCompilation=false
```

期待結果:

- `OfficeItemRentalApp\bin\Debug\net48\OfficeItemRentalApp.exe` が生成される
- ビルドエラーが発生しない

| 備品ID | 備品名 | 区分 |

| P001 | ノートPC-A | PC |
| PJ001 | プロジェクター1 | プロジェクター |

- 田中(営業部)

## 代表的な実施シナリオ

### シナリオA: 登録から返却まで

1. `P001 / ノートPC-A / PC` を登録する
2. `P001` を `田中(営業部)` に貸し出す
3. 備品一覧で `貸出中` と借主が表示されることを確認する
4. 借主検索で `田中` を検索する
5. 備品履歴で `P001` を表示する
6. `P001` を返却する
7. 備品一覧で `利用可` になっていることを確認する
8. 備品履歴で返却日が残っていることを確認する

### シナリオB: 督促対象の確認

1. `PJ001 / プロジェクター1 / プロジェクター` を登録する
2. 返却予定日を今日より前の日付にして貸し出す
3. 備品一覧を表示する
4. 督促列に `督促対象` が表示され、行が薄赤になることを確認する

### シナリオC: 二重貸出防止

1. `O001 / HDMIケーブル / その他` を登録する
2. `O001` を `鈴木(開発部)` に貸し出す
3. 返却前に再度 `O001` を貸し出そうとする
4. 貸出不可となり、現在の借主と返却予定日が表示されることを確認する
